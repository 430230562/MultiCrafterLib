# js,json安装教程

本文为pardon 撰写的 liplum 的多合成工厂库的安装教程,希望对你有帮助.

## 可用安装方式

对于js和json模组,有一下两种配置方式:

### 外部依赖

你需要给你的 `mod.json`或者`mod.hjson`,添加下面这行代码,以确认模组的依赖关系.

```json

"dependencies": ["multi-crafter"]

```

### 内嵌安装

稍微麻烦一点,你需要给你的 `mod.json`或者`mod.hjson`,添加下面这几行代码,来确保java内容被调用.

```json

"java": true,
"main": "net.liplum.MultiCrafterMod",

```

然后解压 `MultiCrafterLib-2.0.0.jar`，将`multicraft/`，`net/`和`classes.dex` 文件放入模组根目录。

解压之前的模组文件结构,假设你模组文件夹名为vne:

- vne/
  - content/
    - crafter.hjson
  - mod.hjson
  - icon.png

解压后应该是这样

- vne/
  - multicrafter/
  - net/
  - content/
    - crafter.hjson
  - mod.hjson
  - icon.png
  - classes.dex

如果你使用js,则需要将 `multicCrafterlib.js` 放入模组的 `scripts/` 目录下。

JS扩展文件：[multicCrafterlib.js](..\injection\assets\scripts\multi-crafter\lib.js)（纯JSON配置无需此文件）

## 工厂定义

安装好后,就可以开始配置工厂了

### 1. JSON 配置方式

如果使用JSON定义工厂，直接在工厂的 `type` 字段填写：

```json

"type": "multicraft.MultiCrafter"

```

#### 配方(`recipes`)的配置

以下是一个正确的配方(`recipes`)配置案例

```json

"recipes": [
  {
    "input": {
      "items": ["copper/2", "lead/1"],
      "fluids": ["water/1.0"],
      "heat": 2,
      "power": 5,
    },
    "output": {
      "items": ["titanium/1"],
      "fluids": ["slag/0.5"]
    },
    "craftTime": 60,
    
    
  },
  {
    "input": {
      "items": ["titanium/1", "silicon/2"],
      "fluids": ["oil/0.8"]
    },
    "output": {
      "items": ["surge-alloy/1"],
      "fluids": ["oil/0.8"],
      "heat": 2,
      "power": 5,
    },
    "craftTime": 120,
    "craftEffect": "smoke"
  }
]
```

每个配方也可单独配置 `icon`、`iconColor` 和 `craftEffect`，实现不同配方的差异化UI展示。

> 注：JSON可用的icon范围有限，JS可通过 `prov(() => Core.atlas.find("liquid-neoplasm"))` 来使用任意贴图以更丰富的自定义绘制，满足复杂界面需求。

其配置逻辑和原生工厂大概相同，但部分接口有关键区别，务必注意以下规则：

- **液体字段**：原生工厂用 `liquids`，本库统一使用 `fluids`，格式为：

  ```json
  "fluids": [
    "water/1.2", "oil/0.8"
  ]
  ```

  数组内格式为 `["液体1/数量","液体2/数量"]`
- **制作时间**：**无全局统一 `craftTime`**，每个配方都有独立的制作时间，必须在单个配方内单独配置
- **输入输出**：物品输入输出格式和原生完全一致，支持多物品、多液体同时配置

#### 多配方分别绘制draw("multicraft.DrawRecipe")

如果你希望不同的配方有不同的Draw，你可以使用`multicraft.DrawRecipe`，下面是参考案例:

```json

"drawer": {
    "type": "multicraft.DrawRecipe"
    "defaultDrawer": 0, //在建筑界面显示的图标
    "drawers": [
        //对应配方一
        {
            //这里面的就和你平常写的相同了
            "type": "DrawMulti"
            "drawers": [
                {
                    "type": "DrawDefault"
                }
                {
                    "type": "DrawFlame"
                    "color": "ffef99"
                }
            ]
        },
        //对应配方二
        {
            "type": "DrawMulti"
            "drawers": [
                {
                    "type": "DrawDefault"
                }
                {
                    "type": "DrawFlame"
                    "color": "ffef99"
                }
            ]
        }
        
    ]
}

```

> 注意defaultDrawer:0是第一个配方的Draw,defaultDrawer:1是第二个配方的Draw。

工厂额外提供 `menu` 接口，用于自定义UI样式，支持4种预设风格：
| 风格类型 | 效果说明 |

|----------|----------|
| `transform` | 带变形动画的交互风格 |
| `simple` | 极简展示风格 |
| `number` | 数字进度展示风格 |
| `detailed` | 详细信息展示风格 |

```json

"menu": "detailed",

```

### 2. JS 编写方式

如果使用 JS 方式定义工厂,需要先导入`multicCrafterlib.js`文件.

```js

const {
    MultiCrafter, DrawRecipe, IOEntry, Recipe
} = require("lib/multicCrafterlib")

```

我们通常这样来创建一个MultiCrafter类的工厂

```js

const cyanidePlant = new MultiCrafter("cyanide-plant");

```

配方编写可具体参考这个文件: [main.js](..\js\assets\scripts\main.js)也可以用js独有的接口:`resolvedRecipes`

```js

  cyanidePlant.resolvedRecipes = Seq.with(
    Object.assign(new Recipe(), {
        input: Object.assign(new IOEntry(), {
            fluids: LiquidStack.with(
            Liquids.cyanogen, 0.05,
            Liquids.neoplasm, 10 / 60),
            power: 1,
            icon: prov(() => Core.atlas.find("liquid-neoplasm"))
        }),
        output: Object.assign(new IOEntry(), {
            items: ItemStack.with(Items.dormantCyst, 1)
        }),
        craftTime: 150.0
    }),
    Object.assign(new Recipe(), {
        input: Object.assign(new IOEntry(), {
            fluids: LiquidStack.with(
            Liquids.cyanogen, 0.05,
            Liquids.neoplasm, 10 / 60),
            power: 1,
            icon: prov(() => Core.atlas.find("liquid-neoplasm"))
        }),
        output: Object.assign(new IOEntry(), {
            items: ItemStack.with(item.protein, 1)
        }),
        craftTime: 50.0
    })
  ),

```

drawer的写法也略有区别:

```js

  cyanidePlant.drawer = Object.assign(new DrawRecipe(), {
```
